# Dify設定メモ

通信代理店向け「店舗スタッフ確認AIデモ」をDifyで構築するための手順書です。

---

## アプリ基本情報

| 項目 | 内容 |
|------|------|
| **アプリ名** | 通信代理店向け 店舗スタッフ確認AIデモ |
| **アプリ種別** | チャットボット |
| **用途** | 営業提案用デモ |
| **対象** | 通信系代理店の営業用 |

---

## 1. Dify で新規アプリを作成

1. Dify にログイン
2. ダッシュボードから「Create a new application」をクリック
3. **アプリケーション名**: `通信代理店向け 店舗スタッフ確認AIデモ`
4. **アプリケーション種別**: `チャットボット` を選択
5. 作成ボタンをクリック

---

## 2. システムプロンプト（指示文）を設定

1. アプリ設定画面を開く
2. **System Prompt** セクションを探す
3. `dify_system_prompt.md` の「システムプロンプト本文」セクションをコピー
4. System Prompt 欄に貼り付け
5. 保存

---

## 3. 知識ベースを追加（RAG設定）

### 3-1. 知識ベース欄を開く

1. アプリ設定 → **Knowledge** または **Knowledge Base** を探す
2. 「Add Knowledge Base」ボタンをクリック

### 3-2. ファイルをアップロード

以下の3ファイルをアップロード：

1. **demo_faq.md**
   - ファイル: `demo_faq.md`
   - 説明: デモ用FAQ（本人確認、未成年契約、法人契約など）

2. **demo_important_notice.md**
   - ファイル: `demo_important_notice.md`
   - 説明: デモ用重要事項説明書（契約前の説明、月額料金、解約条件など）

3. **demo_campaign_terms.md**
   - ファイル: `demo_campaign_terms.md`
   - 説明: デモ用キャンペーン条件表（新規契約キャンペーン、乗り換え、家族割引など）

### 3-3. インデックス作成を待つ

Dify がファイルを解析・インデックス化するまで待機（数分かかる場合があります）。

---

## 4. 会話開始メッセージを設定（オプション）

1. アプリ設定 → **Opening Message** または **Welcome Message**
2. 以下のテキストを設定：

```
通信代理店向け「店舗スタッフ確認AIデモ」です。
料金・規約・キャンペーン・必要書類などについて、デモ資料に記載された範囲で回答します。
資料にない内容は「本部確認が必要」と案内します。
個人情報や実際のお客様情報は入力しないでください。
```

---

## 5. テストを実行

1. チャットウィンドウを開く
2. `dify_test_questions.md` の質問 Q1-Q15 を順番に入力
3. `dify_expected_answers.md` と比較して回答を確認
4. 問題がある場合は、システムプロンプトを修正して再テスト

---

## 6. 公開URL を取得

1. アプリ設定 → **Publish** または **Share**
2. **Make it public** をクリック
3. 公開 URL をコピー（例：`https://dify.ai/chat/...`）

---

## 7. Vercel に環境変数を設定

### 7-1. Vercel ダッシュボードを開く

1. https://vercel.com にログイン
2. `telecom-staff-ai-demo` プロジェクトを選択

### 7-2. 環境変数を追加

1. Settings → **Environment Variables**
2. 新規追加ボタンをクリック
3. **名前**: `NEXT_PUBLIC_DIFY_DEMO_URL`
4. **値**: Dify から取得した公開 URL
5. **Environments**: Production, Preview, Development すべてにチェック
6. Save

### 7-3. 再デプロイ

1. Vercel ダッシュボード → Deployments
2. 最新デプロイの右側 → **...** → **Redeploy** をクリック
3. デプロイ完了を待つ

---

## 8. LP側で動作確認

### 8-1. LP をアクセス

https://telecom-staff-ai-demo.vercel.app/ にアクセス

### 8-2. フォーム送信後の動作確認

1. デモ申込フォームに入力
2. 「送信」ボタンをクリック
3. フォーム送信成功メッセージが表示される
4. **「Difyデモを試す」ボタンが表示される**ことを確認
5. ボタンをクリック → Dify チャットボットが新しいタブで開く

### 8-3. Dify デモテスト

1. 想定質問を入力
2. 期待通りの回答が返される
3. 「本部確認が必要」という案内がある
4. 推測や営業トークがない

---

## トラブルシューティング

### ❌ Dify デモボタンが表示されない場合

**原因**: `NEXT_PUBLIC_DIFY_DEMO_URL` が設定されていない

**対応**:
1. Vercel → Settings → Environment Variables を確認
2. `NEXT_PUBLIC_DIFY_DEMO_URL` が存在するか確認
3. 値が正しい Dify URL か確認
4. Redeploy する

### ❌ Dify デモボタンをクリックしても何も起きない場合

**原因**: Dify URL が正しくない、または Dify が公開されていない

**対応**:
1. Dify ダッシュボード → アプリ → Publish でURL確認
2. URL をコピーしてブラウザで直接アクセス
3. Dify チャットボットが開くか確認
4. URL を修正して Vercel に再設定

### ❌ Dify チャットボットで不適切な回答が返される場合

**原因**: システムプロンプトが不十分、またはファイルインデックスが不完全

**対応**:
1. Dify で System Prompt を確認（`dify_system_prompt.md` と一致するか）
2. 知識ベースファイルがすべてアップロードされているか確認
3. ファイルインデックスが完了しているか確認
4. テスト質問を再実行
5. 回答が改善されない場合は、System Prompt を修正して再テスト

## 注意事項

### 🔒 セキュリティ

- Dify デモに実案件の資料を入れない
- 機密情報を知識ベースに含めない
- 実際のお客様情報の入力を許可しない

### 📝 メンテナンス

- 営業フィードバックを受けたら、System Prompt を改善して再デプロイ
- Dify 側で回答品質が低下したら、ファイルインデックスの再構築を検討
- 定期的に想定質問でテストして動作確認

### 🎯 運用

- Dify デモは「営業用提案デモ」であることを明記
- 「実案件導入時は、御社資料に基づいて再構築する」ことを説明
- デモの限界（個人情報禁止、実料金保証なし、法令判断なし）を周知
