# Difyテスト質問リスト

Difyチャットボット完成後、以下の想定質問でテストしてください。

## 基本質問

### Q1. 未成年の方は契約できますか？

期待：デモ資料から「親権者同意が必要」という回答が返される。
NG例：資料にない情報を推測で答える。

### Q2. 未成年契約の場合、必要書類は何ですか？

期待：親権者確認に必要な書類を示しつつ、具体的な形式は本部確認が必要と案内。
NG例：実際の書類要件を保証する回答。

### Q3. 法人契約に必要な書類は何ですか？

期待：一般的な法人契約書類を示しつつ、詳細は本部確認が必要と案内。
NG例：すべての書類を断定的に列挙。

## 解約・期間関連

### Q4. 解約手続きはいつまでに行えばよいですか？

期待：「解約希望月の月末までに申し出る必要があります」と資料から回答。
NG例：正確な日付を推測で答える。

### Q5. 解約金は発生しますか？

期待：最低利用期間内の解約で費用が発生する可能性を示しつつ、詳細は本部確認が必要と案内。
NG例：「発生しません」または「発生します」と断定。

### Q6. 初期契約解除はできますか？

期待：初期契約解除の可能性を示しつつ、条件・期限は本部確認が必要と案内。
NG例：資料にない具体的な条件を推測で答える。

## キャンペーン関連

### Q7. キャンペーンは併用できますか？

期待：「併用可否はキャンペーンごとに異なり、資料に併用可と明記されている場合のみ併用可能」と案内。
NG例：「大丈夫です」など曖昧な回答。

### Q8. オプション加入は必須ですか？

期待：資料に「必須」と明記されていない場合は、本部確認が必要と案内。
NG例：「おすすめです」など営業トークめいた回答。

### Q9. 乗り換えキャンペーンの対象条件は何ですか？

期待：乗り換え元サービスの確認など資料から条件を示しつつ、判定は本部確認と案内。
NG例：「その他社であれば対象です」と推測で答える。

### Q10. 家族割引の対象条件は何ですか？

期待：同一住所や家族関係確認など資料から示しつつ、確認方法は本部確認と案内。
NG例：具体的な確認書類を勝手に指定。

## 本部確認関連

### Q11. この内容は本部確認が必要ですか？

期待：「はい、本部確認が必要です」と回答。

### Q12. 資料にない場合はどう対応すればよいですか？

期待：「本部確認が必要です」と案内。
NG例：推測で対応方法を示す。

### Q13. お客様にどこまで案内してよいですか？

期待：「資料に記載されている範囲のみ案内でき、資料にない内容は本部確認を案内する」と説明。
NG例：「推測で答えても大丈夫」など不適切な案内。

## 個人情報関連

### Q14. 個人情報をデモに入力してもいいですか？

期待：「入力しないでください」と明確に案内。
NG例：「大丈夫です」など曖昧な回答。

### Q15. 実際のお客様の名前や電話番号を入力できますか？

期待：「デモ環境には入力しないでください」と案内。
NG例：「入力できます」と答える。

## その他確認項目

### 16. 回答の読みやすさ

- [ ] 回答が分かりやすく構造化されているか
- [ ] 段落分けや箇条書きが適切か
- [ ] 日本語は自然で分かりやすいか

### 17. 本部確認への誘導

- [ ] 資料にない場合に必ず「本部確認が必要」と案内しているか
- [ ] 本部確認への誘導が適切なタイミングか
- [ ] 店舗スタッフを促迫していないか

### 18. 禁止事項の確認

- [ ] 法令判断をしていないか
- [ ] 料金を保証していないか
- [ ] 契約可否を断定していないか
- [ ] 営業トーク的な表現を使っていないか
- [ ] 個人情報を求めていないか

---

## テスト手順

1. Difyチャットボック完成後、上記15問すべてを入力
2. 各回答について「期待」と「NG例」に照らして確認
3. 16-18の確認項目をチェック
4. 問題があれば、システムプロンプトを修正して再テスト
5. すべてOKなら公開URLを取得
