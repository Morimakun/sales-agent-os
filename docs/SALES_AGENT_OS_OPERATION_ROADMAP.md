# Sales Agent OS 運用開始ロードマップ

**目的**: AI営業検証ツール（Tier-1）を用いた初期営業検証・モデル検証  
**最重要方針**: X本人に売らない。X は顧客ではなく、紹介候補（Y）を見つけるための橋

---

## 営業モデル（最小化・3層ファネル）

### ビジネスモデル

```
↓  無料・発掘診断（タッチ）
├─ 基本① 「AIで削れる業務 発掘診断」
   価格: 無料～3,000円

↓  有料診断レポート（リード化）
├─ 基本② 「AI業務改善 診断レポート」
   価格: 30,000～50,000円
   （詳細分析＋改善案レポート PDF 納品）

↓  実装PoC（契約・価値確認）
├─ 基本③ 「AI業務改善 ミニPoC」
   価格: 50,000～150,000円
   （スプレッドシート自動化 / FAQ/RAG / 返信支援 など）
```

### X / Y / Z の正確な分類

**X: 自分の既存人脈**
- 友人・知人・既存顧客など、既につながりのある人
- **営業対象ではない**（紹介元として扱う）
- Xに「困っていそうな人がいないか」を聞く
- Xから紹介された人が Y

**Y: 紹介経由の営業候補**
- Xから紹介された見込み客
- 紹介者からの信頼があるため、返信率・相談化率が高い可能性
- **営業対象としては最優先**
- 初回営業では「〇〇さんからの紹介です」と信頼を活用

**Z: 完全新規の営業候補**
- X投稿・クラウドソーシング・Google マップ・Instagram・会社HP・求人ページなどから見つける
- つながりがないため、文面はより丁寧に、売り込み臭を抑える
- **営業対象としては第2優先**（数は取れるが、信頼構築が必要）

**Y軸: 業種・課題カテゴリー**  
- **書類業務系**: 見積書・請求書・契約書（建設・製造・小売）
- **返信業務系**: メール対応・問い合わせ・LINE管理（飲食・美容・EC）
- **管理業務系**: 在庫・日報・スケジュール（多店舗・教育）

**優先度目安（Y / Z の営業対象のみ）**

| 営業対象 | 温度感=高 | 温度感=中 | 温度感=低 |
|--------|----------|----------|----------|
| **Y：紹介経由** | **A+ 即対応** | A 対応 | B 情報収集 |
| **Z：完全新規** | **A 即対応** | B 並列対応 | C 低優先 |

## フェーズ1: 初期検証（5営業日）

### 目的
**仮説の妥当性確認**: 「自分の営業文面で、業種別・困りごと別の反応差を検証できるか」

### 実行プロトコル

#### 1. 候補発見・登録（毎日朝～昼）

Y（紹介経由）候補、クラウドソーシング、Google マップ、Instagram、会社HP、求人ページ、商工会・自治体 DX 情報などから、困っていそうな事業者を見つけます。

**重要**: X本人には売らない。X には「周りで困っていそうな人がいないか」を聞く。営業対象は X ではなく、X の先にいる Y。

**登録手順**:
1. `/opportunities/new` に候補を登録
2. 以下を必ず入力:
   - **候補区分**: Y（紹介経由） / Z（完全新規）
   - **業種**: 飲食・美容・建設など
   - **困っていそうな作業**: メール返信・請求書作成など（仮説で OK）
   - **使っていそうなツール**: Gmail・LINE・Excel など
   - **温度感**: 高 / 中 / 低
3. 「登録して AI 分析へ」をクリック

#### 2. AI分析・文面生成（毎日午後）

**AI分析の確認**:
- 本当に困っていそうか
- 最初に売るべき商品は何か（基本3商品のいずれか）
- 無料相談に誘導できるか

**文面生成**:
- Y（紹介経由）向け初回連絡文
- Z（完全新規）向け初回連絡文
- 無料15分診断オファー文
- 相談後クロージング文（有料診断へ誘導）

**生成文面を人が確認**:
- 売り込み臭くないか
- 相手の業務が自分事になっているか
- 技術用語（AI・DX）を前面に出していないか

#### 3. 返信・相談結果を記録（返信あった後）

Opportunity詳細画面の「メモ」欄に以下を記録：

```
【相談日】2026-06-03
【相手】〇〇さん / △△会社
【業種】飲食
【一番面倒な作業】LINE返信対応
【頻度】毎日30件以上
【使っているツール】LINE, Excel
【AIで軽くできそう】返信の下書き自動生成
【人が確認すべき】返信内容の最終確認（間違いチェック）
【有料診断につながりそう】はい
【次回アクション】見積提示（〇月〇日）
【温度感】高
```

#### 4. 改善サイクル（5件送信後）

5件送信後に結果を分析し、以下を調整：

- **文面**が本当に相手の悩みに寄り添っているか
- **業種別反応**: どの業種が反応しやすいか
- **困りごと別反応**: どの困りごとが刺さるか
- **/settings** の営業トーン・禁止表現を微調整

### 初回5件のKPI

**目的: 売上ではなく、以下を検証する**
- 文面が自然か、売り込み臭くないか
- 「何をしている人か」が伝わるか
- 相手が困りごとを話し始めるか
- **どの業種が反応しやすいか**
- **どの困りごとが刺さるか**

| 項目 | 目標 |
|-----|-----|
| **送信数** | 5 件 |
| **返信数** | 1 件以上あれば成功 |
| **無料相談化** | 0～1 件 |
| **有料診断化** | 0 件でも OK |

**初回送信の目安**:
- Y（紹介経由）: 3 件（優先・信頼度高）
- Z（完全新規）: 2 件（発掘・数は取れる）

### 実績を見ながらの改善（10～20件送信後）

**Y（紹介経由） の場合**:
- 返信率: 20～40%
- 無料相談化: 10～20%

**Z（完全新規） の場合**:
- 返信率: 5～15%
- 無料相談化: 2～10%

これらは「**目指すべき水準**」であり、初回5件で達成する必要はありません。
10～20件の送信と改善サイクルを回しながら、漸進的に改善することを目指します。

### 検証内容（毎送信後、相談ログから確認）

- **文面の自然さ**: 売り込み臭くないか、相手の業務が自分事になっているか
- **業種別反応**: 建設/飲食/小売/EC/士業のうち、どの業種が反応しやすいか
- **困りごと別反応**: 書類業務/返信業務/管理業務のうち、どの困りごとが刺さるか
- **温度感判定の精度**: 「温度感=高」の相手から本当に返信が来るか
- **候補区分の有効性**: Y（紹介経由） vs Z（完全新規） で返信率に差があるか

- [ ] /settings に自社プロフィール入力完了（サービス説明・強み・CTA）
- [ ] /products に 3 商品登録完了

3. **AI プロンプト調整**: 営業トーン・禁止表現・CTA の見直し

## 本システムの制約・ルール

⚠️ **必ず守ること**

1. **送信は手動のみ**
   - システムは「文面候補を生成する」だけ
   - 送信ボタンは存在しない（コピー＆貼り付けで送信）
   - 自動送信・バッチ送信は実装されていない

2. **doNotContact フラグ**
   - 「返信不要」「配信停止」を受けた相手は doNotContact = true に設定
   - これ以降、AI 生成や提案が提示されない（システムが拒否）

3. **候補の多重送信禁止**
   - 同じ相手に異なる提案を短期間に複数送信しない
   - 「応募文」「初回連絡文」「診断オファー」を組み合わせて送らない

4. **スプレッドシートは秘密情報未記入**
   - 個人情報・顧客データ・API キーは保存しない
   - 営業プロセス（日時・提案内容・返信結果）のみ記録

5. **AI が生成した文面は「素材」
   - 必ず 1 度は人間が目を通す
   - 業界用語・製品名・具体的な数字は手直し必須
   - 署名・連絡先は自分で追加

6. **規約・マナー遵守**
   - クラウドソーシング → 応募フォーム使用（DM やコメント欄では送らない）
   - 問い合わせフォーム → 本来の目的（問い合わせ）で使う（営業DM 化しない）
   - SNS → 規約違反（大量自動 DM など）に該当しない方法のみ

## トラブルシューティング

| 問題 | 原因候補 | 対応 |
|-----|-------|-----|
| AI が文面を生成しない | API キー未設定 / 期限切れ | .env.local を確認、APIキー再生成 |
| 相手に「詐欺・怪しい」と言われた | メッセージが長すぎる / 誇大表現 | リスクチェック結果を確認、短く・素直な表現に修正 |
| 返信が来ない（期待より低い） | ターゲット選定が甘い / 真夜中送信 | 温度感「高」のみに絞る、平日9-17時に送信 |
| 案件が「spam」判定された | 相手が未知（信頼スコア低い） | 本名・顔写真・簡単な経歴を署名に追加 |
| 候補が架空・退職者だった | 情報が古い | 候補登録時に「最後に確認した日時」をメモに記入 |

#### 第1候補：美容サロン・整体院

**理由**：
- LINE予約管理がある
- 予約変更対応が頻繁
- 問い合わせ返信が多い
- 口コミ返信対応
- Instagram運用が活発
- 小規模事業者が多く営業対象が見つけやすい

**主な困りごと仮説**：
- LINE予約対応
- 予約変更対応
- 問い合わせ返信
- 口コミ返信
- スタッフ共有
- 毎回同じ説明

#### 第2候補：不動産・保険・士業

**理由**：
- 面談メモが多い
- 次回フォロー業務が必要
- Gmail・LINE返信が頻繁
- 顧客対応履歴が重要
- 相談後のフォローが営業の鍵

**主な困りごと仮説**：
- 面談メモ整理
- 次回フォロー管理
- メール返信下書き
- 顧客対応履歴整理
- 必要書類案内
- FAQ対応

#### 第3候補：紙・PDF・Excel が残る小規模法人

**理由**：
- 高単価化しやすい
- 確認・転記・照合業務が面倒
- AI業務改善と相性がよい

**主な困りごと仮説**：
- 紙帳票確認
- PDF見比べ
- CSV照合
- Excel転記
- FAX確認
- 日報確認
- 目視チェック

### Z営業の禁止事項・運用ルール

#### 絶対禁止

- ❌ 自動一斉送信（100件を同じ文面で送らない）
- ❌ 営業禁止と書かれたフォームに送信
- ❌ 夜中（21時～8時）に大量DM送信
- ❌ 返信がない相手にしつこく追撃
- ❌ 実績を盛る、成果を保証する
- ❌ 「全部自動化できます」と言う
- ❌ 相手の業務を決めつけ説明

#### 厳守ルール

- 1日最大10件送信
- 1業種に絞る（最初は美容で統一）
- 10件ごとに文面を確認・改善
- 送信前に必ず人間が内容確認
- 返信が来たら必ず Opportunity詳細に記録
- 5営業日で50件送ったら必ず振り返り

#### Z向けAI分析で重視する項目

- 完全新規であることを認識
- 売り込み臭を抑える指示
- 業種特有の困りごとに触れる
- 技術ワード（AI、DX）を避ける
- 無料15分相談への自然な導線
- 「もし該当する部分があれば」の柔らかい表現

#### 避けるべき言葉（Z向け文面）

- AIエージェント
- RAG、Supabase、OpenAI API
- Claude Code、Codex
- DX、自動化
- 「何でもできます」

#### 使うべき言葉（Z向け文面）

- 面倒な作業
- 返信、確認、転記、整理、照合
- 人の確認を残す
- 今の業務を変えすぎない
- 小さく試す
- LINE、Gmail、Excel、PDF、紙帳票
