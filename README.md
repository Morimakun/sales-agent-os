# 半自動営業エージェント（Sales Agent OS MVP）

通信系代理店・携帯ショップ運営会社・インターネット回線販売代理店向けに、**店舗スタッフが接客中に社内マニュアル・料金表・FAQ・規約資料を確認できるAIチャットボット**を営業するためのZ営業支援Webアプリです。

このツールは、通信代理店様のニーズを発見し、営業仮説を検証するための支援ツールです。

## 🚀 運用開始ガイド

**すぐに営業検証を始めたい方**は、まず [`docs/SALES_AGENT_OS_OPERATION_ROADMAP.md`](./docs/SALES_AGENT_OS_OPERATION_ROADMAP.md) をご覧ください。

- **無料診断** → **3万円診断レポート** → **ミニPoC（5～15万円）** の 3 層ファネル営業モデル
- **Z完全新規営業検証**: 5営業日・50送信で初期モデル検証（業種絞り・手動確認必須）
- テンプレート・チェックリスト・トラブルシューティング完備

> このツールは **営業準備・文面作成・追客管理を支援** するためのものです。**自動送信機能はありません。** 文面は必ず人間が内容を確認し、手動で送信してください。

## 主役機能（Opportunity・案件候補）

- 案件候補の登録（X投稿 / クラウドソーシング / 求人 / 紹介 / 企業 / イベント / 問い合わせ など）
- AIによる適合度スコア・優先度・課題仮説・推奨オファーの判定
- 応募文 / 初回連絡文 / 紹介依頼文 / 無料診断オファー文 / 追客文 の生成
- 商品メニュー（売る小型商品）の管理
- ダッシュボード・今日のアクション（応募/連絡/紹介/追客/商談化/無料診断候補/受注・失注）
- ワンクリックコピー、ステータス管理、CSV/Markdownエクスポート

## 副次機能（Lead・CSV営業リスト）

- CSVインポート（重複検出つき）、リード一覧・詳細
- リード分析 / 初回営業文 / 問い合わせフォーム文 / 追客文(3/7/14日) / 商談メモ / リスクチェック
- 営業テンプレート管理

## 技術スタック

Next.js 16 (App Router) / TypeScript / Tailwind CSS v4 / Prisma 6 / SQLite / OpenAI または Anthropic Claude API

## セットアップ

```bash
# 1. 依存インストール
npm install

# 2. 環境変数
cp .env.example .env.local
#   .env.local を編集し、使う方のAPIキーを設定
#   AI_PROVIDER=openai なら OPENAI_API_KEY
#   AI_PROVIDER=anthropic なら ANTHROPIC_API_KEY

# 3. DB作成（マイグレーション）
npx prisma migrate dev

# 4. ダミーデータ投入（商品6 / 案件候補8 / リード10 / プロフィール雛形）
npm run seed

# 5. 起動
npm run dev
# http://localhost:3000
```

> APIキーが未設定でも、起動・候補登録・一覧・CSV取込は動作します。**AI分析・文面生成のときだけ** キーが必要です。

## .env.local 設定

| 変数 | 説明 |
| --- | --- |
| `AI_PROVIDER` | `openai`（既定）または `anthropic` |
| `OPENAI_API_KEY` | OpenAI を使う場合 |
| `ANTHROPIC_API_KEY` | Anthropic を使う場合 |
| `AI_MODEL` | 任意。モデルを上書き（既定: openai=`gpt-4o-mini` / anthropic=`claude-haiku-4-5-20251001`） |
| `NEXT_PUBLIC_TELECOM_DEMO_LP_URL` | 通信代理店向けLPのURL。初回営業文では Dify 直URLではなくこちらを案内 |

APIキーは `.env.local` にのみ保存し、Gitにはコミットしないでください（`.gitignore` 済み）。画面・ログにも表示されません。

## CSVフォーマット（リード取込）

ヘッダ行は以下の列名（小文字）。`company_name` は必須、他は任意。

```
company_name,contact_name,email,phone,website_url,contact_form_url,industry,area,source,memo,status
```

サンプル: [`sample-leads.csv`](./sample-leads.csv)。
重複は `company_name + email` / `website_url` / `contact_form_url` で検出し、取込時に除外します。

## 毎日の営業フロー（Z完全新規営業検証モード）

### 初期検証の目標：通信代理店向け営業の当たり筋を見つける

**初回50件（5営業日）の目的**は売上ではなく、以下を確認することです：

- 通信系代理店のどのセグメントが反応するか
- 店舗スタッフの確認業務（料金・規約・キャンペーン・マニュアル確認）の課題が刺さるか
- 問い合わせフォームとメール、どちらの反応がよいか
- 「AI」より「店舗スタッフ向け」「本部問い合わせ削減」などの実務ベース言葉が刺さるか
- 15分の確認業務ヒアリング面談に進む文面はどれか

**初期KPI（目標）**：
- 1日送信: 10件
- 5営業日送信: 50件
- 返信数: 50件中 3～7件を目安
- 無料相談化: 50件中 1～3件を目安
- 有料診断化: 初回50件では0件でも OK

### 1. **業種を1つ選ぶ**

最初は **携帯ショップ運営会社 / 通信代理店** から始めることを推奨します。

理由：
- 店舗スタッフが料金・規約・キャンペーン確認に時間をかけている（課題が明確）
- 複数店舗運営により横展開の余地が大きい
- 本部問い合わせ削減が定量的に判断しやすい
- マニュアル・料金表・FAQ・規約資料が多く、AI活用の効果が出やすい
- 全国に営業対象企業が多い

### 2. **Z候補を探す**

**1日10件、5営業日で50件を探します。**

探す場所：
- Google 検索（「通信代理店 東京」「携帯ショップ 運営」など）
- 採用サイト（求人から複数店舗・スタッフ規模を確認）
- 法人情報サイト（帝国データバンク、ビジネスサーチなど）
- LinkedIn（代理店企業・営業責任者の検索）
- 代理店募集ページ（通信キャリア公式の代理店ネットワーク情報）
- 商工会・自治体情報

**探し方のコツ**：
- 複数店舗運営（2店舗以上）に絞る
- スタッフ求人が出ている企業を優先（確認業務の課題がある証拠）
- 公式な連絡先（問い合わせフォーム、代表メール、採用窓口）がある企業を確認
- 営業連絡禁止の記載がないか確認

### 3. **Z候補を登録**

`/opportunities/new` で10件ずつ登録します。

最小限の情報：
- **会社名**: 〇〇通信代理店
- **URL**: 問い合わせフォーム URL または企業 HP
- **候補区分**: Z（完全新規）
- **紹介元X**: （空欄）
- **業種**: 通信代理店 / 携帯ショップ運営 / インターネット回線販売（統一）
- **困りごと仮説**: 店舗スタッフの料金・規約・キャンペーン・マニュアル確認に時間がかかっている、本部問い合わせが多い、など
- **使っているツール**: PDFマニュアル、料金表、FAQ、Google Drive、Salesforce、Teams など
- **温度感**: 低（デフォルト）

### 4. **AI分析を実行**

登録後、各候補で「AI分析する」をクリック。

AI分析で確認：
- 通信代理店適合度スコア
- 想定される確認業務（料金確認、規約確認、キャンペーン確認など）
- 本部問い合わせ削減余地
- 新人教育支援の余地
- 最初に聞くべき質問（15分のヒアリング質問）
- 推奨初回オファー（無料ヒアリング面談）
- 避けるべき言葉（技術用語、過度なAI推し）

### 5. **文面を生成して確認**

初回メール・問い合わせフォーム用の文面を生成：

- **文面タイプ**: 初回連絡文（通信代理店向け）
- **長さ**: 150～250字
- **トーン**: ビジネス（やや丁寧、営業押しなし）

生成後、必ず人間が確認：
- 「店舗スタッフ向け」「社内確認用」と明記されているか
- 「AI導入」ではなく「15分ヒアリング」から始まっているか
- 相手の業務を決めつけていないか
- 技術ワード（AI、RAG、DX、Supabase など）を使っていないか
- CTA が次の2行になっているか
- `デモ画面のイメージと申込フォームはこちらです。`
- `https://telecom-staff-ai-demo.vercel.app/` または `NEXT_PUBLIC_TELECOM_DEMO_LP_URL` の設定値
- 「デモで小さく試す」「無理な営業はしません」と書いてあるか
- 料金が前面に出ていないか

### 6. **手動で送信**

⚠️ **Sales Agent OS は自動送信しません。** 必ず以下のルール守ってください。

**厳守ルール**：
- 1日最大10件
- 同じ文面をコピペしない（業種ごとに確認）
- 夜中に大量 DM しない
- 返信がない相手にしつこく追撃しない

**送信方法**：
- 問い合わせフォーム → テキストコピーして貼り付け
- Instagram DM → テキストコピーして手動送信
- メール → CC/BCC で一括送信しない（1件ずつ）

### 7. **返信・ヒアリング結果を記録**

返信が来たら、Opportunity 詳細画面の「メモ」欄に必ず記録：

```
【送信日】2026-06-03
【送信チャネル】問い合わせフォーム / メール / 代理店窓口
【返信有無】あり / なし
【返信内容】（相手の返信を引用）
【ヒアリング実施】未実施 / 実施済み
【実際の確認業務課題】（15分ヒアリングで聞いた店舗スタッフの確認業務課題）
【本部問い合わせ削減機会】（どの業務で何割削減できそうか）
【次のアクション】15分ヒアリング実施 / 診断レポート提案 / ミニPoC提案 / 継続フォロー / 終了
【ステージ】初期接触 / ヒアリング中 / 提案中 / 成約 / 失注
【備考】
```

### 8. **10件ごとに文面を改善**

10件送信したら、以下を確認：
- 返信が来た企業の共通点（業種セグメント、店舗数、スタッフ数など）
- 返信が来なかった企業の共通点
- 文面のどの部分（「店舗スタッフ向け」「本部問い合わせ削減」など）が刺さっているか
- 避けるべき言葉は何か（技術用語か、営業押しか）
- 問い合わせフォーム vs メール、どちらの反応がよいか
- ヒアリングで実際に聞いた課題と、AI分析の仮説のズレ

分析結果に基づいて、次の10件の文面を改善。

### 9. **50件送信後に振り返り**

5営業日で50件送ったら、必ず振り返り：

確認項目：
- 業種別返信率（美容の返信率は？）
- チャネル別返信率（DM vs フォーム）
- 無料相談進捗
- 反応が良かった困りごと
- 反応が悪かった言葉
- 次の業種候補（不動産・士業、など）

その上で、方針を調整して次の 50件へ。

## ⚠️ 注意事項（必ずお読みください）

このツールは営業準備・文面作成・追客管理を支援するためのものです。以下の用途では使用しないでください。

- 無差別な営業メール送信
- 事前同意のない広告宣伝メールの大量送信
- 問い合わせフォームへの大量自動送信
- Google Maps等の規約に反するデータ取得
- 送信停止を希望した相手への再連絡
- 虚偽・誇大表現・根拠のない成果保証を含む営業

MVPでは送信機能を実装していません。**人間が内容を確認して手動送信する設計**です。
連絡停止（`do_not_contact`）に設定した相手には、文面生成・連絡操作ができないようにガードしています。
